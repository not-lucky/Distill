# Development

Distill uses a hybrid Node.js (ESM) + Python toolchain. This page lists
the day-to-day commands and the conventions enforced in CI.

## Commands

```bash
npm test              # Run all tests (JS + Python)
npm run test:js       # Run JS tests only
npm run test:py       # Run Python tests only
npm run test:e2e      # JS integration tests only
npm run lint          # ESLint + Prettier + Ruff check + Ruff format check
npm run lint:fix      # Auto-fix lint and formatting issues
npm run format        # Prettier write
npm run format:check  # Prettier check
npm run typecheck     # ty on src/
npm run deadcode      # knip (JS)
npm run deps:check    # depcheck (JS)
npm run duplication   # jscpd
npm run todo:scan     # Tech-debt marker scanner
npm run coverage      # JS coverage report

# Python tools (run via uv):
uv run pytest
uv run ruff check
uv run ruff format
uv run ty check src
uv run vulture src
uv run deptry .
```

## Code conventions

- **Module system:** ESM with `.js` extensions on all relative imports
  (enforced by `import-x/extensions`).
- **Unused variables:** prefix with `_` (e.g. `_error`, `_unused`).
- **Error handling:** always attach `{ cause: error }` when rethrowing
  caught errors.
- **No default exports.** Use named exports throughout.
- **Logging:** use `getLogger(['category'])` from `src/logger.js` and
  prefer tagged-template message interpolation.
- **Console:** only `console.log` and `console.error` are permitted (for
  user-facing CLI output).

## Commit conventions

- `feat:` — new features
- `fix:` — bug fixes
- `refactor:` — code restructuring without behaviour change
- `test:` — test additions or changes
- `chore:` — build, CI, dependency, or tooling changes
- `ci:` — CI/CD pipeline changes
- `docs:` — documentation only

## Tech-debt markers

All `TODO`/`FIXME` markers in source must be annotated with a ticket id:

```js
// TODO(PROJ-123): replace with cached lookup once schema lands
```

`npm run todo:scan` (and the matching `pre-commit` hook) fails the commit
if any unannotated marker is found. See `scripts/scan-todos.js` for the
exact grammar.

## Building the docs

The project ships an MkDocs Material site under `docs/`. To build it
locally:

```bash
uv run mkdocs serve      # local preview on http://127.0.0.1:8000
uv run mkdocs build      # write static site to ./site/
uv run mkdocs build --strict   # fail on any warning (used in CI)
```

The `docs/api/` tree is auto-generated by `mkdocstrings` from docstrings
in the Python source. Treat the source docstrings as the source of
truth — edit Python, not `docs/api/`.

## Pre-commit hooks

JS and Python pre-commit hooks are wired up via:

- `husky` + `lint-staged` (`.husky/pre-commit`) — runs on staged files
  on every `git commit`
- `pre-commit` (`.pre-commit-config.yaml`) — runs the Python side via
  `uv run` so no extra packages are required

Install with:

```bash
npm install                 # installs husky and the prepare script
uv run pre-commit install   # enables the Python hook (requires the
                            # `pre-commit` package — see TODO in
                            # .pre-commit-config.yaml)
```
